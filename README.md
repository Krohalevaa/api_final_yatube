# Проект Yatube
## Описание:
Проект представляет собой REST API для социальной сети Yatube, позволяющей пользователям делиться публикациями, оставлять комментарии, а также подписываться на других пользователей. API предоставляет доступ к операциям CRUD для постов, комментариев и подписок, а также реализует систему аутентификации через JWT.

## Стек технологий:
Python 3.9+
Django 4.0
Django REST Framework 3.13
Simple JWT (для аутентификации)
Djoser (для работы с JWT-токенами)
SQLite

Аутентификация:

В проекте используется аутентификация через JWT-токены. Для получения, обновления и проверки токенов можно использовать следующие эндпоинты:

Получение JWT-токена:
http
Копировать код
POST /api/v1/jwt/create/
Требуется указать логин и пароль для получения токена доступа и обновления.
Обновление JWT-токена:
http
Копировать код
POST /api/v1/jwt/refresh/
Проверка JWT-токена:
http
Копировать код
POST /api/v1/jwt/verify/
Основные эндпоинты API:

Публикации (Posts):
Получение списка публикаций:
http
Копировать код
GET /api/v1/posts/
Параметры:

limit — количество возвращаемых публикаций.
offset — сдвиг относительно начала списка.
Добавление новой публикации:
http
Копировать код
POST /api/v1/posts/
Требуется аутентификация.
Получение публикации по ID:
http
Копировать код
GET /api/v1/posts/{id}/
Обновление публикации:
http
Копировать код
PUT /api/v1/posts/{id}/
Только автор публикации может обновить её. Требуется аутентификация.
Частичное обновление публикации:
http
Копировать код
PATCH /api/v1/posts/{id}/
Только автор публикации может обновить её. Требуется аутентификация.
Удаление публикации:
http
Копировать код
DELETE /api/v1/posts/{id}/
Только автор публикации может удалить её. Требуется аутентификация.
Комментарии (Comments):
Получение списка комментариев к публикации:
http
Копировать код
GET /api/v1/posts/{post_id}/comments/
Добавление нового комментария к публикации:
http
Копировать код
POST /api/v1/posts/{post_id}/comments/
Требуется аутентификация.
Получение комментария по ID:
http
Копировать код
GET /api/v1/posts/{post_id}/comments/{id}/
Обновление комментария:
http
Копировать код
PUT /api/v1/posts/{post_id}/comments/{id}/
Только автор комментария может обновить его. Требуется аутентификация.
Частичное обновление комментария:
http
Копировать код
PATCH /api/v1/posts/{post_id}/comments/{id}/
Только автор комментария может обновить его. Требуется аутентификация.
Удаление комментария:
http
Копировать код
DELETE /api/v1/posts/{post_id}/comments/{id}/
Только автор комментария может удалить его. Требуется аутентификация.
Подписки (Follow):
Получение всех подписок пользователя:
http
Копировать код
GET /api/v1/follow/
Требуется аутентификация. Возможен поиск по параметру search (поиск по подпискам).
Подписка на пользователя:
http
Копировать код
POST /api/v1/follow/
Требуется аутентификация. Подписка на самого себя запрещена.
Сообщества (Groups):
Получение списка сообществ:
http
Копировать код
GET /api/v1/groups/
Получение информации о сообществе по ID:
http
Копировать код
GET /api/v1/groups/{id}/
Примеры запросов:

Получение списка публикаций:
bash
Копировать код
curl -X GET "http://127.0.0.1:8000/api/v1/posts/"
Создание новой публикации:
bash
Копировать код
curl -X POST "http://127.0.0.1:8000/api/v1/posts/" \
-H "Authorization: Bearer <your_token>" \
-d '{"text": "Новая публикация"}'
Подписка на пользователя:
bash
Копировать код
curl -X POST "http://127.0.0.1:8000/api/v1/follow/" \
-H "Authorization: Bearer <your_token>" \
-d '{"following": "user_id"}'

Особенности:

Все эндпоинты, за исключением /follow/, доступны для чтения для анонимных пользователей.
Для создания и управления контентом требуется аутентификация через JWT-токены.
В проекте реализованы кастомные разрешения, которые обеспечивают доступ к изменению и удалению объектов только их авторам.
